# Prompt Gallery 网站开发架构文档（无代码版）

> 本文档为**纯架构/设计文档**，不包含任何代码实现，用于产品规划、技术设计评审与开发协作。
> 目标产品形态参考 prompt gallery / prompt marketplace 类网站（如 localbanana.io），但不涉及任何具体实现细节。

---

## 1. 项目背景与目标

### 1.1 项目背景
随着生成式 AI 的普及，Prompt 已逐渐成为一种可复用、可沉淀、可运营的“数字资产”。  
本项目旨在构建一个 **Prompt 内容聚合与发现平台**，让用户能够：
- 高效浏览和发现优质 Prompt
- 快速复制并使用 Prompt
- 通过榜单与分类理解 Prompt 趋势

### 1.2 核心目标
- 打造一个“以 Prompt 为中心”的内容型网站
- 提供清晰的浏览、筛选、发现体验
- 通过行为数据驱动热门榜单与运营决策
- 为后续的登录、个人库、Prompt Studio 等能力预留扩展空间

---

## 2. 产品范围定义

### 2.1 MVP 范围（第一阶段必须实现）
**公开站功能**
- Prompt 浏览（列表 + 详情）
- 分类导航
- 标签聚合
- 搜索
- 热门榜单（Today / Week / Month）
- Prompt 一键复制

**运营后台**
- Prompt 内容管理（创建 / 编辑 / 发布 / 下架）
- 分类与标签管理（基础能力）

### 2.2 非 MVP（二期及以后）
- 用户系统（登录 / 收藏 / 创作库）
- Prompt 结构化编辑与分析
- 站内生成（接入模型 / Provider）
- 多语言 Prompt 内容
- 商业化（配额、订阅、积分）

---

## 3. 信息架构（IA）

### 3.1 顶层导航结构
- Explore（发现）
  - All
  - TOP（Today / Week / Month）
  - Category（分类）
- Search（搜索）
- Tag（标签聚合）
- About / FAQ / Updates

（登录后）
- Library（个人库）
- Profile（个人资料）

### 3.2 页面职责划分

| 页面 | 职责 |
|----|----|
| Explore / All | Prompt 主发现流 |
| TOP | 热门 Prompt 排行 |
| Category | 分类下 Prompt 聚合 |
| Tag | 标签下 Prompt 聚合 |
| Search | 关键词检索 |
| Prompt Detail | Prompt 详情与复制 |
| Admin | 内容与基础数据管理 |

---

## 4. 系统架构概览

### 4.1 总体架构
- 前端：展示、交互、事件上报
- 后端 API：内容查询、事件接收、榜单数据
- 数据库：内容数据 + 统计数据
- （可选）对象存储：图片、生成结果

### 4.2 核心模块
1. 内容模块（Prompt / Category / Tag）
2. 发现模块（列表 / 搜索 / 榜单）
3. 统计模块（View / Copy）
4. 运营模块（Admin）

---

## 5. 数据架构设计（概念层）

### 5.1 Prompt
- 标题
- Prompt 正文
- 分类（可选）
- 标签（多对多）
- 元信息（模型、比例等）
- 发布状态
- 累计浏览量 / 复制量

### 5.2 Category / Tag
- 名称
- 唯一标识（slug）
- 排序 / 启用状态

### 5.3 行为与统计
- Event：View / Copy 行为
- Daily Stat：按天聚合的统计数据

---

## 6. 热门榜单（TOP）设计

- Today：当天或最近 24 小时
- Week：最近 7 天
- Month：最近 30 天
- 排序指标：Views（默认）/ Copies（可选）

---

## 7. 行为统计与反刷原则

- 为每个访客生成匿名标识
- 同一 Prompt 在时间桶内只记一次行为
- 使用日统计作为榜单数据源
- 防止刷新、连点导致数据失真

---

## 8. 前端架构设计要点

- Sidebar + 内容区布局
- 列表 → 抽屉详情（桌面端）
- 独立详情页（SEO / 移动端）
- 一键复制为核心交互

---

## 9. 后台（Admin）设计要点

- Prompt 生命周期管理
- 分类与标签维护
- 发布与下架控制
- 最小权限原则

## 10. 部署与运行原则

- 配置与代码分离
- 使用托管数据库与 CDN
